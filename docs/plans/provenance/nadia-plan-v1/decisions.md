# nadia-plan: Final Selection Decisions

Decision lead merge of four lens-judge verdict sets (integration-fit, yagni, coordinator-feasibility, plan-quality-value) over the 53 candidates in `/tmp/nadia-plan-build/catalog.md` ┬¦B, against the BASE CONTRACT (┬¦A) and the adoption bar (mandated-by-runtime / closes-documented-gap-cheaply / multi-source-convergence; when in doubt, reject).

Merge rules applied: unanimous verdicts stand; splits decided on the merits with concrete integration-fit/yagni objections beating vague value claims; any base-contract violation is rejected unless the adapt variant removes the violation entirely; same-mechanism candidates merged into one named mechanism; lean beats complete.

Tally: **28 adopt, 17 adapt (all folds/narrowings), 8 reject.** Post-merge the adopted set collapses to **20 named mechanisms** (┬¦2), most of which are prompt text, schema fields, or pure coordinator JS rather than pipeline stages.

## 3. Open-question resolutions (catalog ┬¦C)

1. **Interactive vs autonomous.** Autonomous-first, no interactive blocking anywhere. Mapping: scoping-synthesis confirmation ŌåÆ S2 `adjust-with-logged-assumptions` (b); origin/solo blocking questions ŌåÆ intake classification: truly-blocking ŌåÆ structured halt with question list (c), decidable ŌåÆ hypothesis-answered testable assumption (b); test-strategy human gate ŌåÆ dropped (research folded into S1); depth/appetite ŌåÆ `args.depth` (a); commit consent ŌåÆ `args.commit` (a); N-thing requests ŌåÆ narrow-and-log (b) with exclusions in summary. **Halt contract:** a structured return via M17 (`status:'halted'`, haltStage, haltReason, openQuestions[], draftPath when one exists, roundsUsed) ŌĆö never a throw once intake has run; throws are reserved for pre-flight arg validation. A correctly detected blocker is a successful classification (matches ce-work's late-phase graceful degradation).
2. **Date/NNN.** Optional `args.date` (YYYY-MM-DD); when absent the plan-author agent runs `date` itself. The NNN sequence: the plan-author lists `docs/plans/` itself before writing. (Agents may do I/O; the coordinator never does.)
3. **Location and commit.** `docs/plans/` (ecosystem convention; ce-work fixture uses it; nadia's bare `plans/` loses). `args.commit` DEFAULT FALSE per the global git-read-only rule ŌĆö passing `commit: true` is the consent; either way the run summary tells the caller the next step (commit command hint + the ce-work-deterministic invocation with the fresh planVersion).
4. **Args surface.** Required: `args.request` or `args.origin` (throw early if both missing). Optional: `origin` (path), `depth` ('lightweight'|'standard'|'deep'), `date`, `commit` (default false), `editorRounds` (default 3, clamp 1..5), `reviewRounds` (default 2, clamp 1..3), `spikes !== false`, `externalResearch !== false`. Style mirrors ce-work: `args.X !== false` booleans, `Math.max/min` clamps, single early-throw zone.
5. **Loop composition.** ONE unified round counter ŌĆö the editor loop is the outer (and only) loop, `ROUNDS = Math.max(1, Math.min(5, args.editorRounds || 3))`, `belowBudgetFloor()` at the head. Inside each round, as sequenced stages (never a second counter): personas+primer (rounds 1..`PERSONA_ROUNDS = clamp(args.reviewRounds||2, 1, 3)` only; later rounds are editor-convergence rounds) ŌåÆ JS synthesis ŌåÆ refuters (Ōēż16) ŌåÆ batch-fixer ŌåÆ post-mutation checker + JS checks ŌåÆ fresh editor verdict. This satisfies the orchestrator's editor-outer-with-nested-review shape AND the coordinator-feasibility judge's real concern: there is no loop multiplication (his "sequence, never nest" warning targeted a review-counter ├Ś editor-counter product of ~15 dispatch rounds; a single counter with persona stages that self-disable after PERSONA_ROUNDS has the same bound as his sequenced proposal, ~57 agents worst case in the loop). Exit: READY Ōł¦ blockingCount===0; cap exhaustion ŌåÆ structured halt with unresolved concerns. The spike batch is a boolean-guarded branch, not a loop.
6. **Persona sourcing.** Seven doc-review lenses + research agents via the VERIFIED plugin agentTypes (`compound-engineering:ce-coherence-reviewer`, `ce-feasibility-reviewer`, `ce-product-lens-reviewer`, `ce-design-lens-reviewer`, `ce-security-lens-reviewer`, `ce-scope-guardian-reviewer`, `ce-adversarial-document-reviewer`; `ce-repo-research-analyst`, `ce-learnings-researcher`, `ce-best-practices-researcher`, `ce-web-researcher`, `ce-spec-flow-analyzer`) ŌĆö exactly as ce-work dispatches its code reviewers; re-authoring them locally would fork the evolved review half. NEW local personas only where no plugin equivalent exists: `agents/plan-author.md`, `agents/plan-editor.md`. Reuse `agents/skeptical-refuter.md` with the KTD prompt override (UNVERIFIABLE ŌåÆ Open Questions).
7. **Conditional-persona triggering.** One sonnet classifier agent over the written draft (post-S3), with ce-doc-review's trigger rules ported verbatim into its prompt (incl. the rule that adversarial is NOT triggered by structural complexity or routine validated-origin plans); returns document_type + persona selection, held for all rounds. Mirrors ce-work's riskSurfaces extraction shape.
8. **Research replication depth.** Full ce-plan Phase 1 minus what the registry cannot resolve: always-on `ce-repo-research-analyst` (extended scope: test harness, conventions, CONTEXT.md detection) + `ce-learnings-researcher`; conditional `ce-best-practices-researcher` + `ce-web-researcher` (intent+risk+thin-local gates fed by intake signals, skippable via `args.externalResearch === false`); `ce-spec-flow-analyzer` for Standard/Deep tiers. `ce-framework-docs-researcher` is NOT in the verified registry list ŌåÆ excluded with a `log()` line (no silent cap).
9. **Fix-application authority.** The fixer may apply: safe_auto@anchor-100 findings (base behavior) and fix-now findings that survived refutation. Protected surfaces ŌĆö changes touching Scope Boundaries, the Requirements set, or uid/dependsOn structure are NEVER safe_auto: they require a refutation-surviving finding, and a fix may never widen scope (scope-widening proposals route to Open Questions instead). Every fix verified by M9; conflicts returned unapplied.
10. **Multi-plan output.** Narrow-and-log, always exactly one plan document; exclusions go to Scope Boundaries ŌåÆ Deferred to Follow-Up Work + the run summary's `narrowedScope[]` so the caller can re-invoke per excluded thing. Never N plans (would break ce-work's one-plan-per-invocation contract).
11. **U-ID authority.** The plan-author owns initial assignment. Editor/fixer/spike-revision may ADD `U<max+1>` (gaps fine) or delete, never renumber or reassign; uidŌåÆname identity pairs must not swap. The uid-stability-check is ROUND-FAILING: one explicit re-fix dispatch on violation, then structured halt.
12. **Halt semantics.** Return, don't throw, once past pre-flight arg validation ŌĆö every gate failure, blocking unknown, nonconvergence, and parse failure resolves to the M17 structured not-ready object (draft path included whenever a draft exists). Throws only: missing request/origin, malformed args. Mirrors the sibling's early-hard-guard vs late-graceful split.
13. **"Respect the evolved base" for the document.** The plan document stays byte-compatible with ce-plan's section catalog. NO new sections or fields: no appetite frontmatter, no Size lines, no hill positions, no checkpoints, no parallelization annotations, no mandatory file map, no test-strategy section, no second artifact. Every adopted improvement lands in prompts, agent schemas, coordinator state, gates, or EXISTING sections (## Assumptions, Scope Boundaries, Deferred to Follow-Up Work / Deferred to Implementation, Open Questions). Review-internal artifacts live in the machine-readable run summary, not the doc. (This was also the unanimous implication of all four judges' new-field rejections.)

## 4. Pipeline shape

```
S0 Intake      arg validation (only throw zone) ŌåÆ 1 intake agent
               ŌåÆ Confirmed Intent (locked), {blocking[],decidable[]}, split tests, depth tier
               ŌåÆ blocking unknowns: structured halt return

               Each round (stages, sequenced):
                 a. personas (parallel, Ōēż8) + primer        [rounds 1..clamp(args.reviewRounds||2,1,3) only]
```

### Every loop and its bound

| Loop / fan-out | Bound | Guard |
|---|---|---|
| S4 unified review/editor loop (the only loop counter) | `Math.max(1, Math.min(5, args.editorRounds || 3))`, default 3 | `belowBudgetFloor()` at head; halt-with-concerns at cap |
| Persona stage inside S4 | active rounds 1..`clamp(args.reviewRounds || 2, 1, 3)`; Ōēż8 personas/round | stage self-disables; not a counter |
| Finding refuters | Ōēż16/round, severity-ordered | drops logged ŌåÆ document-as-known-cost |
| Halt-class majority refutation | 3 refuters ├Ś Ōēż3 findings per run | logged |
| KTD refuters | Ōēż8, once (round 1); re-refute only primer-flagged | drops logged |
| Spike batch | once per run (boolean), Ōēż3 spikes + 1 revision pass | runtime unknowns ŌåÆ halt |
| uid-violation re-fix | 1 dispatch, then structured halt | round-failing |
| Parse-gate fix | 1 fixer + 1 re-parse, then structured halt | ŌĆö |
| S5 shared gate-fix | 1 round (boolean) across releasability/origin, then halt | ŌĆö |
| S1 research fan-out | Ōēż6 agents | exclusions/skips logged (incl. ce-framework-docs-researcher) |
| S0/S2/S3/S6 | straight-line, no loops | ŌĆö |

Agent-budget envelope (per the coordinator-feasibility math, adjusted to the unified counter): S0ŌĆōS3 Ōēł 8ŌĆō10; S4 worst case 3 rounds ├Ś ~19 Ōēł 57 (+4 spike branch); S5 Ōēł 5ŌĆō7; S6 Ōēł 2ŌĆō3. **Worst case Ōēł 90, typical Ōēł 45** ŌĆö comfortably inside the 1000-agent ceiling, and bounded by construction at every fan-out.

### Why this is not bloat
The 45-candidate adopted/adapted set collapses to: 1 intake agent, Ōēż6 research agents, 1 gate agent, 1 author, 1 classifier, one bounded loop (personas / refuters / fixer / checker / editor), 3ŌĆō4 tail gates, and a hygiene check. Everything else is prompt text, schema fields, or pure coordinator JS. Every candidate proposing a new document field, second artifact, second loop, or mid-run execution gating was rejected or folded ŌĆö the plan document ships byte-compatible with ce-plan, and ce-work-deterministic's own parser is the release test.
